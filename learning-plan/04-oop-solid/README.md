# 04 — ООП и SOLID

ООП помогает описывать мир с помощью классов.
SOLID — простые правила, чтобы код был гибким.

## Коротко про SOLID
- **S**: класс делает одну работу.
- **O**: расширяем, не ломая старое.
- **L**: наследники ведут себя как базовый класс.
- **I**: не заставляй класс реализовывать лишнее.
- **D**: зависимость от абстракций, а не деталей.

Смотри `oop.py`.
