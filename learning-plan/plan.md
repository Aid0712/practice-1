# Подробный план обучения Python на 3–4 месяца

Ниже — пошаговый путь. Каждый шаг простой и понятный.
Старайтесь учиться 1–2 часа в день, 5–6 дней в неделю.

---

## Месяц 1 — основы языка и мышление программиста

### Неделя 1: старт
- Что такое Python, интерпретатор, файл `.py`.
- Переменные, типы данных: число, строка, булево.
- Простые операции: +, -, *, /, %.
- Ввод/вывод: `input()` и `print()`.

**Результат:** напишете 5–7 маленьких программ.

### Неделя 2: условия и циклы
- `if`, `elif`, `else` — принятие решений.
- Циклы `for` и `while`.
- Ошибки и сообщения об ошибках (простыми словами).

**Результат:** маленький калькулятор и игра «угадай число».

### Неделя 3: коллекции
- Списки, кортежи, множества, словари.
- Базовые методы: добавить, удалить, найти.
- Простой поиск и сортировка.

**Результат:** список задач (to‑do) в памяти.

### Неделя 4: функции
- Как разбивать задачу на кусочки.
- Аргументы, возврат значения.
- Простые правила KISS (делаем проще) и чистого кода.

**Результат:** библиотека полезных функций.

**Результат:** тесты для 3–5 функций.

**Результат:** программа с понятными ошибками.

# Нюансы backend (очень важно)

## 1. Запрос‑ответ
```
Клиент -> Запрос -> Сервер
Сервер -> Ответ -> Клиент
```

## 2. Статусы
- `200` — хорошо
- `400` — ошибка в запросе
- `401` — нет доступа
- `404` — не найдено
- `500` — ошибка на сервере

## 3. Безопасность (база)
- Не хранить пароли в коде.
- Проверять входные данные.
- Логи не должны содержать секреты.

## 4. Данные
- Храним в базе.
- Обновляем через CRUD.
- Делаем резервные копии.

**Подробные объяснения:** смотри `backend-notes.md`.
