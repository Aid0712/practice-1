# Основы backend — простыми словами

Ниже самые важные вещи, которые должен понимать начинающий backend‑разработчик.
Я объясняю как студенту первого курса — простыми и короткими фразами.

---

## 1. Как работает запрос в интернете (HTTP)

**Идея:** клиент просит, сервер отвечает.

```
Клиент (браузер) --> Запрос (request) --> Сервер
Сервер --> Ответ (response) --> Клиент
```

- **Запрос** — «дай мне данные».
- **Ответ** — «вот данные и статус».
- **Статус‑код** — короткое число, которое говорит, что произошло.

Пример статусов:
- `200` — всё хорошо.
- `404` — не найдено.
- `500` — ошибка на сервере.

---

## 2. Что такое API

**API** — это правила общения между программами.
Ты заранее знаешь:
- какой адрес открыть;
- какие данные отправить;
- что получить в ответ.

Пример простого API:
```
POST /notes
{ "text": "Привет" }
```

---

## 3. JSON — формат данных

JSON похож на словарь в Python.

```json
{
  "name": "Аня",
  "age": 18
}
```

---

## 4. База данных и таблицы

**База данных** — это место, где хранятся данные.
**Таблица** — как таблица в Excel.

```
Таблица users
+----+------+-----+
| id | name | age |
+----+------+-----+
|  1 | Аня  |  18 |
+----+------+-----+
```

**CRUD** — 4 действия:
- Create — создать
- Read — прочитать
- Update — обновить
- Delete — удалить

---

## 5. Валидация данных

**Валидация** — проверка входных данных.

Пример простого правила:
- имя не пустое;
- возраст — число.

Это защищает программу от ошибок.

---

## 6. Логирование

**Логи** — записи о том, что делает программа.
Это как дневник приложения.

Зачем?
- найти ошибку;
- понять, что происходило.

---

## 7. Ошибки и исключения

**Исключение** — это ошибка, которая прерывает выполнение.
Мы перехватываем её и даём понятное сообщение.

---

## 8. Безопасность (база)

- Не хранить пароли в коде.
- Использовать переменные окружения.
- Проверять входные данные.

---

## 9. Структура проекта

Простой порядок:
```
project/
  app/
  tests/
  README.md
```

Это помогает не запутаться.

---

## 10. Как запомнить

- **HTTP** = «письмо в интернет».
- **API** = «меню ресторана» (что можно заказать).
- **CRUD** = «Создай‑Прочитай‑Обнови‑Удали».
- **JSON** = «словарь в тексте».
