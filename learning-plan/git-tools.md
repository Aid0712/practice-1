# Git, GitHub и GitLab — простыми словами

## Что такое Git
**Git** — это система контроля версий.
Она помогает сохранять историю изменений в файлах.

Просто говоря:
- Ты работаешь с файлами.
- Git запоминает, что и когда ты изменил.
- Можно вернуться к прошлой версии.

## Что такое GitHub
**GitHub** — это сайт, где можно хранить Git‑репозитории.
Это удобно, чтобы делиться кодом и работать в команде.

Просто говоря:
- Git — это инструмент.
- GitHub — это место в интернете для хранения проектов.

## Что такое GitLab
**GitLab** — похожий сервис, как GitHub.
Там тоже можно хранить репозитории и работать в команде.

## Визуальная аналогия
- Git — это «машина времени» для кода.
- GitHub/GitLab — это «облако», где хранится твой проект.

## Мини‑словарик по Git
- **Репозиторий** — папка проекта с историей изменений.
- **Коммит** — сохранённый «снимок» изменений.
- **Ветка** — отдельная линия разработки.
- **Пуш (push)** — отправка изменений на сервер.
- **Пул (pull)** — получение изменений с сервера.

## Базовые шаги (очень кратко)
1. `git init` — создать новый репозиторий.
2. `git add .` — добавить изменения.
3. `git commit -m "Сообщение"` — сохранить изменения.
4. `git push` — отправить на GitHub/GitLab.

## Как запомнить
- **Commit** = «комментарий о том, что сделал».
- **Push** = «толкнул изменения наверх».
- **Pull** = «подтянул изменения к себе».
